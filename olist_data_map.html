Olist 数据地图

把 11 张表先看成“分组 + 主链路 + 核心连接”的结构图，而不是零散的表名集合。

一、11 张表的分组总览

A 组｜订单主链路

最先掌握

  • olist_orders_dataset：订单主表
  • olist_order_items_dataset：订单商品明细
  • olist_order_payments_dataset：支付明细
  • olist_order_reviews_dataset：评价明细
  • olist_customers_dataset：客户维度
  • olist_products_dataset：商品维度
  • olist_sellers_dataset：卖家维度

这一组的作用：回答订单发生了什么、买了什么、花了多少钱、评价如何、谁买的、谁卖的。

B 组｜辅助维度

第二批掌握

  • product_category_name_translation：品类翻译表
  • olist_geolocation_dataset：地理位置表

这一组的作用：增强解释力，让品类更可读，后续支持州、市、邮编前缀、经纬度等地理分析。

C 组｜扩展业务

先知道，不急着用

  • olist_marketing_qualified_leads_dataset：营销线索
  • olist_closed_deals_dataset：成交线索 / 成交单

这一组的作用：描述销售漏斗世界，和订单履约主链路分开理解更清晰。

三、11 张表的作用卡片

表名 分组 作用 候选主键 核心连接 备注
olist_orders_dataset A 主链 订单主表，记录订单状态和关键时间。 order_id costomer_id、costomers、order_id、tems/payments/reviews 字段类型：已修复；关键空值：
olist_order_items_dataset A 主链 订单商品明细，一单可多行，是最容易导致重复计算的表。 order_id、order_item_id、product_id、seller_id、price、freight_value
olist_order_payments_dataset A 主链 支付方式、分期数、支付金额。 order_id、payment_type、payment_installments、payment_value
olist_order_reviews_dataset A 主链 订单评价与评分。 order_id、review_score、评论时间相关字段
olist_customers_dataset A 主链 客户身份与地区信息。 customer_id、customer_unique_id、customer_city、customer_state
olist_products_dataset A 主链 商品属性与品类。 product_id、product_category_name、重量体积类字段
olist_sellers_dataset A 主链 卖家信息与地区。 seller_id、seller_city、seller_state
product_category_name_translation B 辅助 商品品类翻译映射。 product_category_name、product_category_name_english
olist_geolocation_dataset B 辅助 邮编前缀与经纬度 / 城市 / 州映射。 geolocation_zip_code_prefix、geolocation_lat、geolocation_lng
olist_marketing_qualified_leads_dataset C 扩展 营销线索表，描述潜在客户进入漏斗的阶段。 mql_id、线索来源、创建时间等字段
olist_closed_deals_dataset C 扩展 成交线索 / 成交单，描述漏斗中的成交结果。 mql_id、seller_id、成交时间相关字段

四、核心连接关系

orders.order_id → order_items.order_id
orders.order_id → order_payments.order_id
orders.order_id → order_reviews.order_id
orders.customer_id → customers.customer_id
order_items.product_id → products.product_id
order_items.seller_id → sellers.seller_id
products.product_category_name → product_category_name_translation.product_category_name
customers / sellers 的 zip_code_prefix → geolocation.geolocation_zip_code_prefix

五、使用建议

先只掌握 A 组，再补 B 组，最后再看 C 组。不要一开始就盯所有 column。

更好的做法是先抓三类字段：连接列、指标列、补充解释列。

后续你可以把每张表的详细 column 继续补到这个 HTML 里，做成可展开的数据地图第二版。
